# 1.git配置

```shell
git config --global user.name "用户名"
git config --global user.email "邮箱地址"


# 查看配置信息
git config --global --list
```

# 2.初始化git仓库

```shell
git init [仓库名]
```

# 3.配置本地SSH key

```shell
ssh-keygen -t ed25519 [-C "邮箱地址"]


cat ~/.ssh/id_ed25519.pub
```

配置ssh密钥

# 4.建立远程连接

```shell
# 绑定远程仓库，别名设为 github_ros
git remote add origin git@github.com:xxx
# 重命名分支为main
git branch -m master main
# 建立本地分支和远程分支
git branch --set-upstream-to=origin/main main


# 拉取/推送
git pull
git push
```

## 常见问题

拉取时出现无效引用：fatal: 无效引用：origin/main

```shell
# 获取远程分支信息，不会改变本地数据
git fetch origin

#查看分支信息
git branch -r
```

更新指定文件

```shell
git checkout origin/main -- <文件名>
```



执行 `git pull origin main` 时遇到「fatal: 需要指定如何调和偏离的分支」错误，核心原因是：**本地分支和远程分支的提交历史出现了 “分叉”（偏离）**，Git 不知道该用「合并（merge）」还是「变基（rebase）」的方式整合这两段偏离的历史，因此需要你明确指定整合策略。

可配置 Git 全局默认用「合并」方式处理偏离分支，后续所有仓库的 pull 都会按这个规则来。

```shell
# 设置全局默认用合并（merge）方式 pull
git config --global pull.rebase false
```
