# git branch

主分支永远是稳定、可发布的版本，你写新功能、改 bug 都在**自己的分支**上做，不会污染主线。

**常见分支模型**

- `main/master`: 主分支

- `dev`: 开发分支

- `feature/xxx`: 新功能分支

- `bugfix/xxx`: bug修复分支

## 查看当前分支

```shell
git branch
```

带`*`号就是当前所在分支

## 创建新分支并切换分支

```shell
git branch <分支名>
git checkout <分支名>
# git 2,23以上
git switch <分支名>

# 上面两个命令可以合并成下面这个命令
git checkout -b <分支名>
# git 2.23使用下面命令
git switch -c <分支名>
```

## 版本切换

### 查看历史版本

```shell
git log
git log --oneline
```

`commit 914cba0440593a60ee31724037b5516d581c2409`前7位是版本号，版本号为914cba0。

```shell
# 切换到某个历史版本
git checkout 914cba0
```

**注意：** 切换版本号后处于游离HEAD状态，如果要进行修改，需要新建分支。

```shell
git checkout -b <新分支名> <历史版本号>
```

## 分支合并

切换回主分支，合并分支

```shell
git checkout main
git merge <分支名>
```
